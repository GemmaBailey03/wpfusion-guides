MemberPress + CRM: membership access via tags

Updated: 30 June 2026

MemberPress handles membership levels, subscriptions, and gated content on WordPress. Your CRM (ActiveCampaign, Keap, HubSpot, Mailchimp, and others) holds contacts, tags, and email automation. Without a bridge, you manually export members, rebuild segments in your CRM, or maintain separate Zapier workflows every time a subscription changes.

WP Fusion connects MemberPress to 60+ CRMs: when someone joins, renews, upgrades, cancels, or fails a payment, WP Fusion creates or updates the contact and applies tags you define. You can also restrict WordPress content based on CRM tags — so membership status in your CRM drives what members see on your site.

The problem with disconnected membership and CRM data

  • Stale segments — your CRM still shows “Active Member” after a subscription expires
  • Manual onboarding — new members do not enter welcome sequences until someone exports a CSV
  • No CRM-driven access — you cannot unlock bonus content from a tag applied in an email automation
  • Fragmented automations — churn, trial conversion, and upgrade events live in MemberPress but not in your CRM

What WP Fusion syncs from MemberPress to your CRM

See the official MemberPress documentation for full settings. Typical sync includes:

  • Contact records — create or update CRM contacts at checkout with custom registration fields
  • Membership tags — apply tags when a level is purchased, renewed, upgraded, downgraded, paused, or cancelled
  • Subscription events — separate tags for trial signup, trial conversion, payment failure, and refund
  • Membership fields — sync level name, status, expiration date, subscription total, and lifetime value to CRM custom fields
  • Auto-enrollment — optionally link a CRM tag to a membership level so applying the tag grants access (and removing it revokes access)

How setup works (high level)

  1. Install WP Fusion on WordPress and connect your CRM (see the getting started guide).
  2. Edit each MemberPress membership level → open the WP Fusion tab.
  3. Set Apply Tags for registration and configure tags for subscription cancelled, paused, upgraded, downgraded, payment failed, trial, and refund events.
  4. Optionally enable Remove Tags so registration tags drop when a membership expires.
  5. Map MemberPress fields under WP Fusion → Contact Fields (membership level, expiration date, subscription status, etc.).
  6. Build CRM automations triggered by those tags — welcome sequences, renewal reminders, win-back offers.

Restrict WordPress content using CRM tags

WP Fusion can gate pages, posts, and MemberPress Courses based on tags in your CRM — not just MemberPress roles. On any post or page, use the WP Fusion meta box to set Required tags (any) or Required tags (all). Users without the tag see a restricted message or redirect to a sales page.

This works well for bonus modules, alumni-only content, or upsell pages unlocked after a CRM automation applies a tag. Full details: restricting access to content.

MemberPress supports multiple concurrent memberships, so tag changes are tied to subscription and transaction events rather than a single “current level.” WP Fusion applies tags when members sign up, when subscription status changes, when transactions expire, and on upgrade/downgrade — see the MemberPress docs for the full event list.

MemberPress Courses and abandoned checkout

If you use MemberPress Courses, WP Fusion can tag members when they complete a course, lesson, or quiz. Configure tags on each course, lesson, or quiz alongside MemberPress access rules.

With a Plus or Professional WP Fusion license, the Abandoned Cart addon can capture checkout details before purchase completes and sync them to your CRM for recovery campaigns. Tags are set on the WP Fusion tab when editing each membership level.

MemberPress + CRM vs Zapier

Zapier works for simple one-way triggers (new membership → add tag). WP Fusion is stronger when you need:

  • Bi-directional field sync between WordPress user meta and CRM custom fields
  • Tag-based content restriction on pages, courses, and downloads
  • Many MemberPress events (trial conversion, payment failure, upgrade/downgrade) without separate Zaps
  • CRM tag → membership auto-enrollment via linked tags and webhooks
  • One contact record when the same person buys from WooCommerce, learns in LearnDash, and holds a MemberPress subscription

Course creators may also use LearnDash + ActiveCampaign tagging alongside MemberPress Courses.

Setup checklist

  1. Connect your CRM in WP Fusion.
  2. Configure tags on each MemberPress membership level (registration + subscription events).
  3. Map membership level, expiration date, and subscription status to CRM fields.
  4. Set required tags on protected content or MemberPress Courses.
  5. Run a test signup on staging → confirm contact, tags, and fields in your CRM.
  6. Optionally run batch operations under WP Fusion → Advanced to retroactively tag existing members.
